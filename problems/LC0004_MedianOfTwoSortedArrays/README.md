# LC0004 — Median of Two Sorted Arrays

LeetCode: https://leetcode.com/problems/median-of-two-sorted-arrays/

Сложность: Hard  
Теги: array, binary-search, divide-and-conquer

Кратко:
Даны два отсортированных массива целых чисел nums1 и nums2. Нужно найти медиану объединённого массива за время лучшее чем O(m+n), желательно O(log(min(m,n))).

Подход (описание):
- Производим бинарный поиск по меньшему из двух массивов (пусть nums1 короче).
- Идея — разбить оба массива на "левую" и "правую" части так, чтобы в левой части было ровно half = (m+n+1)/2 элементов.
- Пусть i — число элементов из nums1 в левой части, j = half - i — из nums2.
- Для корректного разбиения должны выполняться:
    Aleft <= Bright  и  Bleft <= Aright
  где Aleft = nums1[i-1] (или -inf, если i==0), Aright = nums1[i] (или +inf, если i==m),
        Bleft = nums2[j-1] (или -inf), Bright = nums2[j] (или +inf).
- Если условие верно, медиана:
    - при нечетной суммарной длине: max(Aleft, Bleft)
    - при четной длине: (max(Aleft,Bleft) + min(Aright,Bright)) / 2
- Иначе смещаем границу поиска: если Aleft > Bright — уменьшаем i, иначе увеличиваем i.

Сложность:
- Время: O(log(min(m,n)))
- Память: O(1)

Особенности реализации:
- Используем INT_MIN и INT_MAX для крайних случаев (когда i==0 или i==m и т.д.).
- Всегда бинарим по меньшему массиву, чтобы логарифм был по меньшему размеру.
- Код корректно обрабатывает случаи, когда один массив пуст.

Примеры:
- nums1 = [1,3], nums2 = [2] → медиана = 2.0
- nums1 = [1,2], nums2 = [3,4] → медиана = 2.5
