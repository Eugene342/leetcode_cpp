# LC0027 — Remove Element

LeetCode: https://leetcode.com/problems/remove-element/

Difficulty: Easy  
Tags: array, two-pointers (in-place), std::algorithm

Кратко:
Дан массив `nums` и значение `val`. Удалить все вхождения `val` "на месте" (in-place) и вернуть новую длину `k` так, чтобы первые `k` элементов массива содержали все элементы, отличные от `val`. Порядок элементов среди оставшихся не важен.

Подход:
- Используется стандартная функция `std::remove` (алгоритм "erase–remove idiom" без непосредственного вызова erase): `it = std::remove(nums.begin(), nums.end(), val)`, затем `k = distance(nums.begin(), it)`.
- `std::remove` сдвигает все элементы != val в начало диапазона и возвращает итератор на новый логический конец.
- Сложность: время O(n), дополнительная память O(1).

Граничные случаи:
- Пустой массив -> вернуть 0.
- Если val не встречается -> вернуть исходную длину.
- Если все элементы равны val -> вернуть 0.

Пример (self-test в `remove_element.cpp`):
- Вход: nums = [3,2,2,3], val = 3 → k = 2, первые k элементы != 3.
- Вход: nums = [0,1,2,2,3,0,4,2], val = 2 → k = 5.

Интеграция:
- Файл `remove_element.cpp` содержит `main()`, который выполняет несколько тестов и возвращает 0 при успехе — удобно для запуска с `ctest`.
- CMake в корне автоматически подхватит новую папку и создаст таргет с именем `LC0027_RemoveElement` (имя папки используется как имя таргета).
